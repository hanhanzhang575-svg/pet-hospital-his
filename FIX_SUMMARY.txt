================================================================================
FastAPI 服务启动问题 - 完整诊断和解决方案
================================================================================

【诊断时间】2024年
【项目】白之助宠物医院信息管理系统 (Shironosuke Pet Hospital IS)
【问题】http://127.0.0.1:8000/docs 无法访问

================================================================================
【问题根因】
================================================================================

SQLITE3 DLL 加载错误:
  - ImportError: DLL load failed while importing _sqlite3
  - 原因: 虚拟环境 (.venv) 基于 Miniconda Python 3.9.12
  - 该版本与当前系统 Python 3.13.4 库不兼容
  - 导致 SQLAlchemy 无法导入，数据库初始化失败
  - 最终 FastAPI 应用无法启动

环境信息:
  系统 Python: C:\Users\zhangmohan\AppData\Local\Programs\Python\Python313\python.exe (3.13.4)
  虚拟环境: C:\Users\zhangmohan\Desktop\信息系统\.venv (Miniconda Python 3.9.12)
  项目路径: C:\Users\zhangmohan\Desktop\信息系统

================================================================================
【5分钟快速解决方案】(推荐)
================================================================================

方案: 使用系统 Python，不修改虚拟环境

步骤 1: 打开命令提示符 (cmd.exe)
  - 按 Win+R
  - 输入 cmd
  - 按回车

步骤 2: 进入项目目录
  cd /d C:\Users\zhangmohan\Desktop\信息系统

步骤 3: 安装依赖（一行命令，复制粘贴）
  python -m pip install fastapi uvicorn sqlalchemy python-jose passlib apscheduler networkx chromadb httpx python-multipart

步骤 4: 启动 FastAPI 服务（一行命令，复制粘贴）
  python -m uvicorn backend.main:app --host 127.0.0.1 --port 8000

步骤 5: 验证
  - 您应该看到: Uvicorn running on http://127.0.0.1:8000
  - 打开浏览器访问: http://127.0.0.1:8000/docs
  - 看到蓝色 Swagger UI 文档页面表示成功
